SEV-SNP ID Block Builder

Builds an ID block for SEV-SNP guest launch. The ID block contains guest policy, metadata (family_id, image_id, guest_svn), and a placeholder launch digest. Calculating launch digests, ID block signing and ID auth info structure are out of scope of this simple tool.

Guest Policy (ID block offset 0x58)

8 bytes. Byte 0 (lowest address) is ABI minor, byte 1 is ABI major, bytes 2–7 are policy flags. Written to memory as a 64-bit little-endian unsigned integer.

ABI Version (bits 15:0)
Minimum SEV-SNP firmware ABI version required for guest launch. The firmware rejects the launch if its version is below this value (fw_error=7). Setting major=0 minor=0 accepts any firmware version. Test platform: API 1.54.
Policy Flags (bits 63:16)
Bits 63:16 of the policy. Default 00000000000b = the most permissive value possible based on applicable (ABI version) AMD documentation, on configuration of one compliant test system and on behavior of the SW on this system, between the test SW and the SNP FW (eg, KVM). This SW can and in some cases does block certain values from making it to the FW.
Enter as big-endian hex: leftmost digit is bits 63:60.
Bit 17 (MBO) is cleared — KVM will reject with EINVAL.
Reserved bits set (18 or 21-63) — KVM will reject with EINVAL.
Full 64-bit policy = (flags << 16) | (abi_major << 8) | abi_minor.
